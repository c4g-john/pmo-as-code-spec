# Security Policy

This repository holds a specification document, so it carries little direct
security risk. Report anything you consider a security concern (for example a
change that would weaken the conformance or blocking semantics in a way tools
would inherit) by email to **john@c4genterprises.com** rather than in a public
issue.

For vulnerabilities in the reference implementation, see the security policy in
[c4g-john/docassert](https://github.com/c4g-john/docassert/security/policy).
